# Thema: portmapper/nfs vulnerability
## Übung ssh auf Metasploitable
### Angabe: 
Als Vorbereitung für die folgende Übung.   
Konfiguriere passwortloses ssh Login auf Metasploitable mit Hilfe eines ssh Public/Private keys.  
Betrachte besonders die notwendigen Änderungen in `~/.ssh/authorized_keys`, diese Datei muss in der nachfolgenden Übung manipuliert werden.   

Hinweis: der ssh-server auf Metasploitable unterstützt nur Schlüssel im RSA Format (nicht die neueren default Formate wie ed255…).
Der Kali ssh Client lehnt die Verbindung mit diesem alten Schlüssel standardmäßig ab, aber man kann es mit Optionen wieder erzwingen.
Bei ssh Verbindungsproblemen hilft häufig die Verwendung der Verbose Optionen des ssh clients (-v, -vv oder -vvv).

### Umsetzung 
#### RSA-Key erzeugen (auf Kali)
Befehl:   
```
ssh-keygen -t rsa -b 2048 -m PEM
```

![](Uhttps://github.com/user-attachments/assets/5d9fcdcc-872c-4985-ad27-34d43f84b552)    
Der key wird ins File "metasploitable_rsa" gespeichert. (Datei metasploitable_rsa für private key und Datei metasploitable_rsa.pub für public key).   

#### Public Key auf Metasploitable bringen
Zuerst mit normalen Passwort einloggen:   
Befehl: `ssh msfadmin@192.168.83.133` funktioniert nicht aufgrund eines Verbindungsproblems.   

Befehl, der funktioniert:   
```
ssh -o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
msfadmin@192.168.83.133
```
Mit diesem Befehl wird ein rsa-Key akzeptiert.

SSH-Verzeichnis vorbereiten (auf Metasploitable):   
Befehle:   
```
mkdir -p ~/.ssh   
chmod 700 ~/.ssh
```
Man muss die Rechte auf 700 setzen, damit ssh die Dateien akzeptiert.    

Public Key einfügen:    
Auf Kali:   
```
cat ~/.ssh/metasploitable_rsa.pub
```
Mit diesem Befehl den kompletten Inhalt kopieren (der Befehl muss im Kali ohne ssh Verbindung ausgeführt werden)   

Auf Metasploitable (im Kali ausführen, wenn per ssh auf Metasploitable verbunden):    
```
vi ~/.ssh/authorized_keys
```
Mit diesem Befehl Public Key einfügen.

Dann mit `chmod 600 ~/.ssh/authorized_keys` Berechtigungen setzen. 

#### Prüfen
nun sollte man sich mit dem Befehl:   
```
ssh -i ~/.ssh/metasploitable_rsa \
-o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
msfadmin@192.168.83.133
```
per ssh auf Metasploitable verbinden können. -> funktioniert noch nicht

#### Fehlerbehebung
Ich habe den public key aus der Datei im Kali erneut kopiert und erneut in die authorized_keys Datei im Metasploitable eingefügt. Jetzt funktionierts.

## Übung: portmapper/nfs
### Angabe:
Metasploitable 2 enthält eine Schwachstelle im Zusammenhang mit portmapper/nfs.

- Informiere dich grob über nfs (network file system) und den Zusammenhang mit portmapper   
- Was kannst du über nmap und nmap scripts über die nfs Schwachstelle herausfinden?   
- Nutze die Schwachstelle aus indem du über das File-System einen ssh key in das Homeverzeichnis von root hineinschwindelst. Das Ziel ist, dass vom Angreifer-System aus ein passwortloses SSH login auf root@metasploitable möglich ist.

### Recherche
NFS (Network File System) ist ein Protokoll, das es erlaubt, Verzeichnisse über das Netzwerk freizugeben und wie lokale Ordner einzubinden. Der Server stellt Ordner bereit, der Client „mountet“ sie.

Portmapper / rpcbind ist ein Dienst auf Port 111, der bei älteren NFS-Versionen Clients sagt, auf welchem dynamischen Port der NFS-Dienst läuft. Ohne ihn wüsste der Client nicht, wohin er sich verbinden soll.

Sicherheitsrelevanz:   
- Offene NFS-Freigaben können ausgenutzt werden (z. B. Dateien manipulieren, SSH-Keys platzieren).
- `no_root_squash` ermöglicht Root-Zugriff auf Client-Seite – sehr unsicher.
- Mit Tools wie `showmount -e <IP>` lassen sich freigegebene Verzeichnisse anzeigen.

Ports:   
- NFS: 2049
- Portmapper/rpcbind: 111
